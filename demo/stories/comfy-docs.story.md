# ComfyUI Docs — User Story

## Who
A new ComfyUI user who just installed it and wants to build their first workflow. They have a blank canvas and no idea where to start.

## Goal
Find and follow a getting-started tutorial to create a text-to-image workflow, gaining confidence to explore further on their own.

## Happy path

1. **Land on the docs home** — Open docs.comfy.org. The hero section immediately signals this is the official documentation hub for ComfyUI.
2. **Explore sidebar navigation** — The sidebar organizes content by skill level. Getting Started guides are right at the top — exactly where a new user needs them.
3. **Open the search bar** — Have a specific question (e.g., ControlNet). Use the search to jump straight to the answer instead of browsing manually.
4. **Review search results** — Results show matching pages across all sections — tutorials, node references, and developer guides, all relevant to the query.
5. **Browse the getting started section** — Each tutorial walks through workflows step by step with screenshots and downloadable workflow files.
6. **Check the node reference** — Every built-in node documented with parameters, examples, and tips. This is where power users live.
7. **Notice quality-of-life features** — Dark mode, responsive layout, and clean typography keep reading comfortable day or night.

## Key features to demo
- Sidebar navigation organized by skill level
- Search functionality (Ctrl+K or search button)
- Getting started tutorials with step-by-step guidance
- Node reference documentation
- Dark mode toggle
- Clean, scannable page layout

## Implicit narration (what the voiceover should explain)
- WHY the docs site matters (it's the fastest path from "confused" to "creating")
- HOW the sidebar is structured (skill-level progression)
- WHAT search can do (jump to any topic instantly)
- WHERE to go next (getting started for beginners, node reference for power users)

## Out of scope for this demo
- Actually running ComfyUI (that's the frontend demo)
- Custom node development (that's a developer-focused demo)
- API reference deep-dive (too technical for this audience)
