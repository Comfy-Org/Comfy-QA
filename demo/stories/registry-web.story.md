# Comfy Registry — User Story

## Who
A ComfyUI hobbyist or professional who just discovered they need a custom node — maybe an LLM router, a new sampler, or a video model — and wants to find a vetted package they can install in their workflow.

## Goal
Discover, evaluate, and copy the install command for a custom node pack from a trusted central place.

## Happy path

1. **Land on the registry home** — They arrive from a Discord link or a Google search. The hero says "Welcome to ComfyUI Registry" and immediately makes it clear this is the official place.
2. **Notice the search bar** — Front and center. Placeholder reads "Search Nodes".
3. **Type a query** — They type something like "controlnet" because that's the most common reason people come here.
4. **See the result** — Search suggestions appear as they type. They scan repository names, version numbers, and download counts.
5. **Click into a node** — They open the most popular result to see the full details page.
6. **Read the README** — The README has install instructions, screenshots, and dependencies.
7. **Copy the install command** — One click copy so they can paste it into their ComfyUI manager.
8. **Browse versions** — They check version history to make sure the node is actively maintained.
9. **Get back to browsing** — Maybe they look for a second related node before leaving.

## Key features to demo
- The hero / branding
- The search bar (the single most-used feature)
- Live search suggestions
- Detail page with README
- Install command (the conversion event)
- Version history (the "is this maintained?" check)
- The login/signup nav for publishers

## Implicit narration (what the voiceover should explain)
- WHY a registry exists (without it, you'd be installing random GitHub repos and hoping)
- WHO publishes here (verified node authors)
- HOW it integrates with ComfyUI Manager
- WHAT to do next (link to docs)

## Out of scope for this demo
- Logging in / publishing your own node (that's a separate "publisher" demo)
- The admin / moderation views
- The CI / verification badge details
